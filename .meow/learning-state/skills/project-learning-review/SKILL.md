# Project Learning Review

1. Read `.meow/learning-state/constitution.md` and `.meow/learning-state/current-state.md`.
2. Identify the project outcome, constraint, or non-goal affected by the request.
3. Separate verified evidence, owner-confirmed facts, inferred patterns, and unavailable evidence.
4. Prefer the smallest change that advances an outcome or prevents a verified failure.
5. When intention is missing or conflicting, ask one question and explain which decision it unlocks.
6. Present new rules, skills, memories, or workflows as learning proposals. Do not activate them without owner approval.
7. Include verification and rollback evidence for consequential changes.
